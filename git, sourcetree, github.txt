working directory : .git이 존재하는 폴더
 -> 해당 폴더에서 변경되는 모든 내용을 추적

staging(stage) area : ( 스테이지에 올라간 파일 영역)
기록하려는 파일이 머무는 곳(이곳에 올라온 파일들을 기록 할 예정이다)

[commit]
staging area에 올라온 파일을 local repository에 저장하는(맡기는) 명령어

local(현재 컴퓨터의) repository(저장소) : 현재 컴퓨터의 저장소에 변경 내용을 저장할 저장소
 - local : 내 컴퓨터라는 의미를 가지고 있다.

커밋 되돌리기
- revert : 커밋 되돌리기
- reset : 선택한 커밋까지 현재 브런치를 초기화

2. github 로그인

3. 원격 저장소(remote repository) 페이지로 이동

4. github CODE버튼 클릭 -> clone -> 주소 복사

5. source tree 새탭 -> clone버튼 

6. 소스경로에 4번서 복사한 주소 붙여넣기

7. 목적지 경로 설정(로컬파일)

8. 선택한 위치 뒤에 \폴더명을 추가 작성하여 새 폴더를 만들어 정리 가능
